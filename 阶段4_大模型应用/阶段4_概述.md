# 阶段4：大模型与前沿应用（3-4个月）

## 阶段目标
掌握大语言模型(LLM)原理和应用开发，能够构建基于大模型的智能应用。

---

## 什么是大语言模型？

**一句话**：大语言模型是"读了海量文本学会说话"的AI，能理解和生成人类语言。

**类比**：
```
传统AI：教小孩做数学题（规则明确）
大模型：教小孩读万卷书后写文章（从数据中学习）

ChatGPT = 读了互联网上几乎所有文字后学会的"超级学生"
```

**为什么叫"大"模型？**
- 参数量大：GPT-3有1750亿参数
- 训练数据大：几十TB文本
- 计算资源大：需要成千上万GPU

## 学习内容概览

### 1. 大语言模型基础（4-5周）

**核心问题**：LLM是怎么工作的？

| 概念 | 通俗解释 |
|------|----------|
| 预训练 | 先读海量文本，学会语言规律 |
| 微调 | 在特定任务上进一步学习 |
| Tokenization | 把文字拆成小块（token） |
| 注意力机制 | 每个词关注其他所有词 |
| 自回归 | 一个一个词地生成 |

### 2. Prompt工程（3-4周）

**核心问题**：如何让LLM更好地完成任务？

| 技巧 | 通俗解释 |
|------|----------|
| 零样本 | 直接问，不给例子 |
| 少样本 | 给几个例子再问 |
| 思维链 | 让模型一步步思考 |
| 角色扮演 | 设定AI的身份 |
| 系统提示 | 设定AI的行为规则 |

### 3. 微调技术（4-5周）

**核心问题**：如何让通用模型变成专业模型？

| 方法 | 通俗解释 |
|------|----------|
| 全参数微调 | 调整所有参数（耗资源） |
| LoRA | 只调一小部分参数（省资源） |
| 指令微调 | 教模型理解指令 |
| RLHF | 让人类反馈指导学习 |
| DPO | 直接学习人类偏好 |

### 4. RAG应用开发（3-4周）

**核心问题**：如何让LLM访问私有知识？

| 概念 | 通俗解释 |
|------|----------|
| RAG | 先检索相关文档，再生成答案 |
| 向量数据库 | 存储文档的"含义向量" |
| 文档分块 | 把长文档切成小块 |
| 重排序 | 对检索结果重新排序 |

### 5. Agent开发（4-5周）

**核心问题**：如何让LLM使用工具完成任务？

| 概念 | 通俗解释 |
|------|----------|
| Agent | 能思考、行动的AI助手 |
| 工具调用 | 让AI使用计算器、搜索等 |
| 规划 | 让AI制定执行计划 |
| 记忆 | 让AI记住对话历史 |
| 多Agent | 多个AI协作完成任务 |
