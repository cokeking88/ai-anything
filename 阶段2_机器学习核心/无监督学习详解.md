## 学习目标
掌握无监督学习核心算法，理解聚类、降维、异常检测的原理和应用场景。

## 无监督学习概述

**一句话解释**：无监督学习就是让机器自己在数据中找规律，没有人告诉它什么是正确答案。

**生活类比**：
- **监督学习**：老师教你认动物（这是猫，这是狗）
- **无监督学习**：给你一堆动物照片，让你自己分类（你发现有些长耳朵，有些短耳朵）

**主要任务**：
- **聚类**：把相似的东西分到一组（客户分群）
- **降维**：把复杂的数据变简单（压缩信息）
- **异常检测**：找出与众不同的点（欺诈检测）

## 第一周：聚类算法

### 1.1 K-Means聚类详解

#### 什么是K-Means？

**直观理解**：
想象你是一家奶茶店的老板，想在全国开几家分店。你会：
1. 先随便选K个位置开分店
2. 看看每个顾客离哪个分店最近，归到那个分店
3. 根据每个分店的顾客位置，把分店挪到顾客的中心位置
4. 重复2-3，直到分店位置不再变化

```
初始：随机选3个中心点
    ·    ★    ·
      ·      ★
    ·    ·    ★

迭代后：中心点移到顾客中心
      ★···
     ·   ·★··
      ··  ·★
```

**正式定义**：
K-Means通过迭代优化，将数据划分为K个簇，使每个点到它所属簇中心的距离之和最小。

#### 核心思想：两个步骤交替

**步骤1：分配（Assignment）**
```
对于每个数据点：
    计算到所有中心点的距离
    分配到最近的那个中心点
```

**步骤2：更新（Update）**
```
对于每个中心点：
    移动到所属所有点的平均位置
```

**可视化理解**：
```
第1轮：          第2轮：          第3轮：
  ·  ★  ·        ·★    ·         ★··
    ·  ·    →     ·  ★  ·    →    ·★·
  ·  ★  ·        ·  ·  ★         ··★

★代表中心点，·代表数据点
中心点逐渐移动到最佳位置
```

#### 如何选择K值？

**方法1：肘部法则**

画出K值与"总距离"的图，找拐点：

```
总距离
  |  \
  |   \
  |    \___
  |        \____
  +--------------→ K值
      ↑
    拐点（选这个K）
```

**方法2：轮廓系数**

衡量每个点与自己簇的紧密程度和与其他簇的分离程度：
- 轮廓系数 = 1：完美聚类
- 轮廓系数 = 0：在边界上
- 轮廓系数 < 0：可能分错类了

#### K-Means的优缺点

**优点**：
- ✅ 简单易懂
- ✅ 速度快，适合大数据
- ✅ 效果通常不错

**缺点**：
- ❌ 需要预先指定K
- ❌ 只能发现圆形/球形簇
- ❌ 对初始中心点敏感
- ❌ 容易受异常值影响

#### 现实应用

| 场景 | 应用 |
|------|------|
| 电商 | 客户分群（高消费、低频次、新客户等） |
| 图像 | 图像分割、颜色量化 |
| 文档 | 文档聚类、主题发现 |
| 地理 | 选址分析、配送中心规划 |

#### Python实现
```python
from sklearn.cluster import KMeans
import numpy as np

# 客户分群示例
# 特征：消费金额、消费频次
X = np.array([
    [100, 5],   # 低消费低频次
    [500, 20],  # 高消费高频次
    [120, 3],   # 低消费低频次
    [480, 18],  # 高消费高频次
    # ... 更多数据
])

# 聚类
kmeans = KMeans(n_clusters=2, random_state=42)
labels = kmeans.fit_predict(X)

print(f"聚类结果: {labels}")
print(f"簇中心: {kmeans.cluster_centers_}")
```

### 1.2 层次聚类详解

#### 什么是层次聚类？

**直观理解**：
想象你在整理家族族谱：
1. 先把每个人当作一个单独的家族
2. 把最亲近的两个人合并成一个小家族
3. 再把最亲近的两个小家族合并
4. 重复，直到所有人合并成一个大家族

```
           全家族
           /    \
         /        \
      小家族1    小家族2
      /    \      /    \
    A      B    C      D
```

**正式定义**：
层次聚类通过逐步合并或分裂簇，构建一个层次结构（树状图）。

#### 两种方法

**凝聚型（自底向上）- 最常用**：
```
开始：A  B  C  D  E（每个点是一个簇）
第1步：[A,B]  C  D  E（合并最近的A和B）
第2步：[A,B]  [C,D]  E
第3步：[A,B,C,D]  E
第4步：[A,B,C,D,E]（所有点合并）
```

**分裂型（自顶向下）**：
```
开始：[A,B,C,D,E]（所有点在一个簇）
第1步：[A,B] [C,D,E]（分成两组）
第2步：[A] [B] [C,D] [E]
...（继续分裂）
```

#### 如何衡量两个簇的距离？

```
方法1：单链接（最近距离）
  簇A中最近的点  ←→  簇B中最近的点

方法2：全链接（最远距离）
  簇A中最远的点  ←→  簇B中最远的点

方法3：平均链接
  簇A所有点  ←→  簇B所有点 的平均距离
```

#### 树状图（Dendrogram）

树状图展示了聚类的完整过程：

```
距离
  |        ___________
  |       |
  |    ___|       ___
  |   |   |      |
  | __|   |    __|
  ||  |   |   |  |
  A  B   C   D  E
```

**如何确定K？**
- 在某个高度"切一刀"，穿过几条线就是几个簇

#### 层次聚类的优缺点

**优点**：
- ✅ 不需要预先指定K
- ✅ 可以得到层次结构
- ✅ 可以用树状图可视化

**缺点**：
- ❌ 计算复杂度高（O(n³)）
- ❌ 不适合大数据集
- ❌ 一旦合并就不能撤销

#### 现实应用

| 场景 | 应用 |
|------|------|
| 生物 | 物种分类、基因表达分析 |
| 文档 | 文档层次分类 |
| 营销 | 客户层级细分 |
| 图像 | 图像分割 |

#### Python实现
```python
from sklearn.cluster import AgglomerativeClustering
from scipy.cluster.hierarchy import dendrogram, linkage
import matplotlib.pyplot as plt

# 层次聚类
hierarchical = AgglomerativeClustering(n_clusters=3)
labels = hierarchical.fit_predict(X)

# 绘制树状图
Z = linkage(X, method='ward')
plt.figure(figsize=(10, 5))
dendrogram(Z)
plt.title('层次聚类树状图')
plt.show()
```

### 1.3 DBSCAN详解

#### 什么是DBSCAN？

**直观理解**：
想象你在地图上找城市：
- **城市核心**：人口密集的地方（周围有很多人）
- **郊区**：在城市边缘（离城市近但人不多）
- **荒野**：前不着村后不着店（周围没人）

DBSCAN就是找出这些"人口密集"的区域。

```
    ·  ···
   ··· ·★· ·        ★ = 核心点（密集）
    ····· ·          · = 边缘点
         ·    ×      × = 噪声点（孤立）
```

**正式定义**：
DBSCAN基于密度进行聚类，能找到任意形状的簇，并能识别噪声点。

#### 核心概念

**ε（eps）邻域**：
- 以点p为中心，半径ε的圆内的所有点

**核心点**：
- ε邻域内至少有MinPts个点
- 相当于"城市中心"

**边界点**：
- 在某个核心点的ε邻域内，但自己不是核心点
- 相当于"郊区"

**噪声点**：
- 既不是核心点也不是边界点
- 相当于"荒野"

```
核心点 ★：周围≥MinPts个点
边界点 ○：在核心点周围，但自己不够
噪声点 ×：孤零零的
```

#### 算法流程

```
1. 随机选一个点
2. 检查它是不是核心点
   - 是：创建新簇，把它和周围ε内的点都加入
   - 否：暂时跳过
3. 对于新加入的点，如果是核心点，也把它的邻居加入
4. 重复直到所有点都被访问
5. 没被任何簇包含的点就是噪声
```

#### 参数选择

**ε（半径）选择**：
```
画出每个点到第k个邻居的距离排序图
距离
  |          ___________
  |        /
  |      /
  |    /
  |___/
  +----------------→ 点
       ↑
     拐点 = 最佳ε
```

**MinPts选择**：
- 经验值：MinPts = 2 × 特征维度
- 或者 MinPts = 维度 + 1

#### DBSCAN的优缺点

**优点**：
- ✅ 不需要预先指定簇数
- ✅ 能发现任意形状的簇
- ✅ 能识别噪声点
- ✅ 对异常值不敏感

**缺点**：
- ❌ 对参数ε和MinPts敏感
- ❌ 对密度不均匀的数据效果差
- ❌ 高维数据效果不好

#### 现实应用

| 场景 | 应用 |
|------|------|
| 地理 | 城市聚类、热点区域发现 |
| 安全 | 入侵检测、异常行为识别 |
| 图像 | 图像分割 |
| 社交 | 社区发现 |

#### Python实现
```python
from sklearn.cluster import DBSCAN

# 异常检测示例
dbscan = DBSCAN(eps=0.5, min_samples=5)
labels = dbscan.fit_predict(X)

# labels = -1 表示噪声点
n_clusters = len(set(labels)) - (1 if -1 in labels else 0)
n_noise = list(labels).count(-1)

print(f"发现的簇数: {n_clusters}")
print(f"噪声点数: {n_noise}")
```

## 第二周：降维技术

### 2.1 PCA（主成分分析）详解

#### 什么是PCA？

**直观理解**：
想象你在给一群人拍照：
- **侧面照**：能看到身高差异（信息多）
- **头顶照**：只能看到头大小（信息少）

PCA就是找到"信息最多"的拍照角度，把高维数据压缩到低维。

```
原始数据（2D）         PCA后（1D）
    ·   ·                  · ·
  ·   ·   ·      →       · · · ·
    ·   ·                  · ·
```

**正式定义**：
PCA找到数据方差最大的方向（主成分），将数据投影到这些方向上，实现降维。

#### 核心思想：方差最大 = 信息最多

**为什么要最大化方差？**
- 方差大 = 数据分散 = 保留了更多信息
- 方差小 = 数据聚集 = 信息少

```
方向1（方差大）：     方向2（方差小）：
  ·       ·              · · ·
    ·   ·                  ·
  ·       ·              · · ·
  ←————————→            ←——→
  信息多                 信息少
```

#### 算法步骤

```
1. 中心化：每个特征减去均值
2. 计算协方差矩阵：描述特征之间的关系
3. 特征值分解：找到主成分方向
4. 选择前d个主成分：保留大部分信息
5. 投影：把数据映射到低维空间
```

#### 方差解释率

```
第1个主成分：解释60%的方差
第2个主成分：解释25%的方差
第3个主成分：解释10%的方差
...
累积：95% → 只需要3个主成分就够了
```

#### PCA的优缺点

**优点**：
- ✅ 简单快速
- ✅ 可以去除噪声
- ✅ 便于可视化（2D/3D）
- ✅ 可以加速后续算法

**缺点**：
- ❌ 只能捕捉线性关系
- ❌ 主成分可能难以解释
- ❌ 对异常值敏感

#### 现实应用

| 场景 | 应用 |
|------|------|
| 图像 | 人脸识别、图像压缩 |
| 数据预处理 | 特征降维、去除冗余 |
| 可视化 | 高维数据可视化 |
| 去噪 | 信号处理、图像去噪 |

#### Python实现
```python
from sklearn.decomposition import PCA

# 降到2维
pca = PCA(n_components=2)
X_2d = pca.fit_transform(X)

# 查看方差解释率
print(f"方差解释率: {pca.explained_variance_ratio_}")

# 选择保留95%方差的维度数
pca_full = PCA()
pca_full.fit(X)
cumsum = np.cumsum(pca_full.explained_variance_ratio_)
d = np.argmax(cumsum >= 0.95) + 1
print(f"保留95%方差需要 {d} 个主成分")
```

### 2.2 t-SNE详解

#### 什么是t-SNE？

**直观理解**：
想象你有一张皱巴巴的世界地图（高维），你想把它摊平（2D）看。t-SNE会尽量保持原来相邻的国家还是相邻，即使形状会变形。

```
高维空间（皱巴巴的）    t-SNE后（摊平的）
    ·★· ·                  ★ ★
  · ·★· · ·       →       · · ·
    · ·★·                    ·
```

**正式定义**：
t-SNE是一种非线性降维方法，特别适合高维数据可视化，能保留数据的局部结构。

#### 核心思想：保持邻居关系

**高维空间**：
- 用高斯分布计算点与点的相似度
- 相近的点相似度高

**低维空间**：
- 用t分布计算点与点的相似度
- 优化目标：让低维的相似度尽量接近高维的相似度

#### t-SNE的优缺点

**优点**：
- ✅ 可视化效果非常好
- ✅ 能保留局部结构
- ✅ 能发现非线性结构

**缺点**：
- ❌ 计算很慢
- ❌ 结果每次可能不同
- ❌ 只能用于可视化
- ❌ 不同参数结果差异大

#### 现实应用

| 场景 | 应用 |
|------|------|
| 图像 | 可视化图像特征 |
| 文本 | 词向量可视化 |
| 生物 | 单细胞数据可视化 |
| 探索性分析 | 数据探索 |

#### Python实现
```python
from sklearn.manifold import TSNE

# t-SNE降维
tsne = TSNE(n_components=2, perplexity=30, random_state=42)
X_tsne = tsne.fit_transform(X)

# 可视化
plt.scatter(X_tsne[:, 0], X_tsne[:, 1], c=y, cmap='tab10')
plt.title('t-SNE可视化')
plt.show()
```

### 2.3 UMAP详解

#### 什么是UMAP？

**直观理解**：
UMAP是t-SNE的升级版，更快、更好，还能保持全局结构。

```
t-SNE：                    UMAP：
  只能看到局部关系            能看到全局关系
  ·····                       ·····
  ·★···    ←局部紧密           ·★···   ←局部紧密
  ·····                       ·   ·
  ·····                       ·····   ←全局也有结构
```

#### UMAP vs t-SNE

| 特性 | t-SNE | UMAP |
|------|-------|------|
| 速度 | 慢 | 快5-10倍 |
| 全局结构 | 保留差 | 保留好 |
| 可复现性 | 差 | 好 |
| 可用于新数据 | 否 | 是 |

#### Python实现
```python
# pip install umap-learn
import umap

# UMAP降维
reducer = umap.UMAP(n_components=2, random_state=42)
X_umap = reducer.fit_transform(X)

# 可视化
plt.scatter(X_umap[:, 0], X_umap[:, 1], c=y, cmap='tab10')
plt.title('UMAP可视化')
plt.show()
```

## 第三周：异常检测

### 3.1 什么是异常检测？

**直观理解**：
异常检测就是找出"与众不同的点"：
- 信用卡盗刷（消费模式异常）
- 工业设备故障（传感器数据异常）
- 网络入侵（流量模式异常）

```
正常数据：········
           ······
            ·····
异常点：        ×（离群很远）
```

### 3.2 Isolation Forest详解

#### 什么是Isolation Forest？

**直观理解**：
想象你在一堆人中找"特殊的人"。你随机问问题：
- "身高>180？" → 分成两组
- "体重>80kg？" → 继续分

**异常的人很快就被单独分出来了**（因为太特别了），而正常人需要问很多问题才能分开。

```
正常点：需要很多次划分才能隔离
异常点：很少次划分就被隔离了

            划分1
           /    \
         正常    划分2
                /    \
             正常   异常（很快隔离）
```

**正式定义**：
Isolation Forest通过随机划分数据来"隔离"异常点，异常点通常需要更少的划分次数。

#### 核心思想

**异常分数**：
- 路径短 → 异常分数高 → 可能是异常
- 路径长 → 异常分数低 → 可能是正常

#### Isolation Forest的优缺点

**优点**：
- ✅ 速度快，适合大数据
- ✅ 不需要假设数据分布
- ✅ 适合高维数据

**缺点**：
- ❌ 对局部异常不敏感
- ❌ 需要设置污染率

#### Python实现
```python
from sklearn.ensemble import IsolationForest

# 异常检测
iso_forest = IsolationForest(contamination=0.1, random_state=42)
labels = iso_forest.fit_predict(X)

# labels = 1 正常，labels = -1 异常
n_outliers = (labels == -1).sum()
print(f"检测到 {n_outliers} 个异常点")
```

### 3.3 LOF（局部异常因子）详解

#### 什么是LOF？

**直观理解**：
LOF比较一个点和它邻居的"拥挤程度"：
- 如果你和邻居一样拥挤 → 正常
- 如果你比邻居稀疏很多 → 可能是异常

```
正常区域（密集）：
  ·····
  ·····
  ·····

异常点（稀疏）：
         ×（离大家很远）
```

**正式定义**：
LOF通过比较点的局部密度与其邻居的局部密度来检测异常点。

#### LOF分数

- **LOF ≈ 1**：正常（和邻居一样密集）
- **LOF > 1**：可能是异常（比邻居稀疏）
- **LOF >> 1**：很可能是异常（非常稀疏）

#### LOF vs Isolation Forest

| 特性 | Isolation Forest | LOF |
|------|------------------|-----|
| 速度 | 快 | 慢 |
| 全局异常 | 擅长 | 不擅长 |
| 局部异常 | 不擅长 | 擅长 |
| 高维数据 | 好 | 差 |

#### Python实现
```python
from sklearn.neighbors import LocalOutlierFactor

# LOF异常检测
lof = LocalOutlierFactor(n_neighbors=20, contamination=0.1)
labels = lof.fit_predict(X)

# labels = 1 正常，labels = -1 异常
n_outliers = (labels == -1).sum()
print(f"检测到 {n_outliers} 个异常点")
```

## 算法选择指南

### 聚类算法选择

```
需要聚类
  │
  ├── 知道要分几类？
  │     ├── 是 → K-Means
  │     └── 否 → 继续判断
  │
  ├── 数据有噪声？
  │     ├── 是 → DBSCAN
  │     └── 否 → 层次聚类
  │
  └── 数据量大？
        ├── 是 → K-Means
        └── 否 → 层次聚类
```

### 降维算法选择

```
需要降维
  │
  ├── 目的是什么？
  │     ├── 可视化 → t-SNE / UMAP
  │     ├── 加速算法 → PCA
  │     └── 去噪 → PCA
  │
  └── 数据有非线性结构？
        ├── 是 → t-SNE / UMAP
        └── 否 → PCA
```

### 异常检测算法选择

```
需要异常检测
  │
  ├── 异常类型？
  │     ├── 全局异常 → Isolation Forest
  │     └── 局部异常 → LOF
  │
  └── 数据量大？
        ├── 是 → Isolation Forest
        └── 否 → LOF
```

### 算法对比总结

| 类别 | 算法 | 优点 | 缺点 |
|------|------|------|------|
| 聚类 | K-Means | 简单快速 | 需要K，球形簇 |
| 聚类 | 层次聚类 | 层次结构 | 慢，不适合大数据 |
| 聚类 | DBSCAN | 任意形状，去噪 | 参数敏感 |
| 降维 | PCA | 快，可解释 | 只能线性 |
| 降维 | t-SNE | 可视化好 | 慢，只用于可视化 |
| 降维 | UMAP | 快，全局结构 | 较新 |
| 异常 | Isolation Forest | 快，高维好 | 局部异常差 |
| 异常 | LOF | 局部异常好 | 慢，高维差 |
